# Деплой CharSheet

Жодного ручного «скидання локальних даних» більше не потрібно — додаток сам
перевіряє токен при старті й тихо виходить, якщо той протух. Старий кеш
очищається автоматично через версіонування persist.

## Як це працює

Бекенд (Fastify) **сам віддає зібраний фронт**. Тобто фронт і API живуть на
**одному порту й домені** — немає CORS, немає vite-проксі, відносні `/api`
шляхи працюють скрізь (і локально, і на хості).

```
react-v2/
  dist/            ← зібраний фронт (npm run build)
  backend/         ← Fastify; якщо бачить ../dist — роздає його
```

## Локальний запуск (один порт, як на проді)

```bash
cd react-v2
npm run install:all     # ставить залежності фронту + бекенду
npm run build           # збирає фронт у dist/
cd backend
JWT_SECRET=якийсь_довгий_секрет npm start
# → http://localhost:3000  (і фронт, і API)
```

Або однією командою з кореня:

```bash
JWT_SECRET=секрет npm start   # build + запуск бекенду
```

## Локальна розробка (два процеси, з hot-reload)

```bash
# термінал 1
cd react-v2/backend && JWT_SECRET=секрет npm run dev   # :3000

# термінал 2
cd react-v2 && npm run dev                              # :5173 (vite-проксі на :3000)
```

У dev-режимі фронт на :5173, а vite проксує `/api` на :3000.
Коли `dist/` немає — бекенд це розуміє й працює тільки як API.

## Деплой на хост (Railway / Render / VPS / Docker)

Потрібен **один** сервіс — Node.js, що запускає `backend/server.js`.

### Змінні оточення
| Змінна | Значення |
|--------|----------|
| `JWT_SECRET` | довгий випадковий рядок (обов'язково в проді) |
| `NODE_ENV` | `production` |
| `PORT` | хост зазвичай задає сам |
| `DB_PATH` | (опц.) шлях до файлу БД, напр. `/data/charsheet.db` на постійному диску |

### Команди
```
Build:  npm run install:all && npm run build
Start:  node backend/server.js
```

(Або просто `npm start` з кореня — він робить build і запускає бекенд.)

### Важливо про базу
SQLite зберігається у файл (`charsheet.db`). На платформах з ефемерною
файловою системою (Railway/Render без диску) база **обнуляється при кожному
рестарті**. Підключи постійний диск (volume) і вкажи `DB_PATH` на нього.

### SSE за реверс-проксі (nginx)
Якщо ставиш nginx перед додатком — для живих оновлень вимкни буферизацію:

```nginx
location /api/events {
    proxy_pass http://localhost:3000;
    proxy_set_header Connection '';
    proxy_http_version 1.1;
    proxy_buffering off;
    proxy_cache off;
    proxy_read_timeout 24h;
}
```

(Сам бекенд уже шле `X-Accel-Buffering: no`, тож часто цього достатньо.)

## Окремий хостинг фронту й бекенду (якщо все ж потрібно)

Якщо фронт деплоїш окремо (Vercel/Netlify), а бекенд окремо — постав у
`.env.production` фронту:

```
VITE_API_URL=https://твій-бекенд.домен
```

і на бекенді в `CORS_ORIGIN` додай домен фронту. Але простіший і
рекомендований шлях — один сервіс, як описано вище.
